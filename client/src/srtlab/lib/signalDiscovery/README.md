# Signal Discovery (Task #625)

A port of the **TUMFTM "Holistic Approach for Automated Reverse Engineering
of UDS Data"** methodology to SRT Lab.

- Paper:   *Battisti, Reiter, Lienkamp* ‚Äî "A Holistic Approach for
  Automated Reverse Engineering of UDS Data on Electric Vehicles."
  *World Electric Vehicle Journal* 16(7):384, 2025.
  DOI: <https://doi.org/10.3390/wevj16070384>
- Upstream code: <https://github.com/TUMFTM/holistic-uds-reverse-engineering>
- License:       Apache-2.0 (see upstream `LICENSE`).

## Workflows

| Pane    | Purpose                                                            |
|---------|--------------------------------------------------------------------|
| Sweep   | Enumerate live ECUs and discoverable DIDs on the bus.              |
| Record  | Drive the car; log DID samples + ground-truth OBD-II PIDs.         |
| Match   | Brute-force candidate decoders, correlate vs. PIDs, label the DID. |

## Files

- `decoder.js` ‚Äî pure candidate decoders (u8/i8/u16/i16/u32/i32 BE/LE),
  hex helpers, Pearson correlation, OLS regression, and `bestCandidate()`
  that returns the best (decoder, byteOffset) for a (samples, ground-truth)
  pair plus its r¬≤, slope, intercept.
- `sweepEngine.js` ‚Äî async-generator `sweepDidRange()` and a `probeEcu()`
  helper that drive the existing `@workspace/uds` builders/parsers through
  whatever transport (`initAdapter` / `createEngineForActiveTransport`).
- `__tests__/decoder.test.js` ‚Äî 15 vitest cases covering byte-order, sign
  extension, correlation, regression, and an end-to-end `bestCandidate`
  against a synthetic OBD-II coolant-temperature shape.

## Persistence

All persistence is via the API server at `/api/signal-discovery/*`:

- `POST   /sweeps`                          create a sweep
- `GET    /sweeps?vin=‚Ä¶`                    list (per VIN)
- `GET    /sweeps/{id}`                     fetch sweep + ECUs + DIDs
- `PATCH  /sweeps/{id}`                     update status / cursor
- `DELETE /sweeps/{id}`                     remove
- `POST   /sweeps/{id}/ecus`                upsert discovered ECUs
- `POST   /sweeps/{id}/dids`                upsert discovered DIDs
- `POST   /experiments`                     create a recording experiment
- `GET    /experiments?vin=‚Ä¶`               list
- `GET    /experiments/{id}`                fetch + samples
- `PATCH  /experiments/{id}`                start/stop
- `DELETE /experiments/{id}`                remove
- `POST   /experiments/{id}/samples`        append batched samples
- `GET    /catalog?vin=‚Ä¶`                   list labelled DIDs
- `POST   /catalog`                         upsert a catalog entry

OpenAPI definitions live in `lib/api-spec/openapi.yaml`; React Query hooks
and Zod schemas are generated by `pnpm --filter @workspace/api-spec run codegen`.

## Out of scope (per task spec)

- DoIP transport, neural-network matching, Python upstream wholesale port.
- Security access during sweep ‚Äî sweeps run in default session only.
- Any change to `tools/python-bridge/` or the J2534 bridge protocol.
