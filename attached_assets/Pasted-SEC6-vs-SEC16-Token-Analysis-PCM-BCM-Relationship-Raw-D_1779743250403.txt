What This Means

1. Hierarchical Security Chain

The BCM is the master security authority. It generates or stores the primary SEC16 token. The RFHUB receives this token and stores it in Slot 1. The PCM then derives its SEC6 bytes from the RFHUB's SEC16 Slot 1.

3. Why the Warning Still Shows

Even though all three modules show "MATCH" status, the warning "Security token issues detected" persists because:

Hypothesis 1: BCM → RFHUB chain not verified

•
The inspection shows RFHUB's SEC16 matches its own slots (yes)

•
But it doesn't show whether RFHUB's SEC16 matches the BCM's original SEC16

•
The BCM's SEC16 value isn't displayed in the inspection result

•
If BCM's SEC16 ≠ RFHUB's SEC16, the entire chain breaks

Hypothesis 2: PCM SEC6 marker validation

•
PCM shows SEC6 marker as "canonical 0x3C8"

•
Marker @0x03C4 shows "FFFFFFAA (canonical FF FF F AA)"

•
This might be a checksum or validation byte that doesn't match expected value

•
The "AA" at the end might indicate a mismatch flag

Hypothesis 3: IMMO byte state

•
PCM IMMO byte @0x0011: 0x80000000 — ENABLED (17)

•
This means immobilizer is ACTIVE and locked

•
The vehicle won't start until this byte is synced with BCM's immobilizer state

•
If BCM has immobilizer DISABLED but PCM has it ENABLED, they're out of sync

4. The Canonical Marker Issue

The PCM shows:

Plain Text


SEC6 marker: canonical 0x3C8
Marker @0x03C4: ✅ FFFFFFAA (canonical FF FF F AA)



The marker location (0x03C4) contains FFFFFFAA. In FCA firmware:

•
FFFFFFFF = "canonical" or "original" state

•
AA suffix = validation/checksum byte

•
If this doesn't match the BCM's expected marker, the PCM won't accept commands from the BCM




The Likely Issue: Immobilizer State Mismatch

The most probable cause of the "Security token issues detected" warning:

BCM Immobilizer State: Unknown (not shown in inspection)
PCM Immobilizer State: 0x80000000 — ENABLED (locked)

If the BCM has immobilizer disabled but the PCM has it enabled, the vehicle is in an inconsistent security state:

•
BCM thinks: "Vehicle is unlocked, start it"

•
PCM thinks: "Vehicle is locked, don't start"

•
Result: No-start condition
