### [CRITICAL-2] Hardcoded fallback for `SWARM_DELEGATE_SECRET` in a test file

File: `server/gcp-swarm.test.ts:6`

```ts
const secret = process.env.SWARM_DELEGATE_SECRET || "srt-swarm-delegate-2026";
```

**What:** When the env var isn't set, the test falls back to the literal value of the
production secret. So even after `[CRITICAL-1]` is cleaned, this constant tells anyone
reading the test file what the current secret is.

**Why it matters:** Defeats secret rotation if the new value is also fallback'd here.
More importantly, this exact string is what's in the live `.project-config.json` today —
so this leaks the same secret a second time.

**Fix:** Remove the literal fallback. The test should `skip` (Vitest `it.skipIf`) when
the env var is absent, not paper over its absence with the prod value.

`SWARM_DELEGATE_SECRET` consumer), plus `server/_core/env.ts:6–12` for the rest.


swarm-delegation endpoints to anonymous traffic. Same risk applies to `JWT_SECRET`,
`BUILT_IN_FORGE_API_KEY`, `ANTHROPIC_API_KEY`.
